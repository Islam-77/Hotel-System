<------------------Hotel Management System – Classes Descriptions------------------>

This project is a Simple Hotel Management System developed in Java. It allows hotel staff to manage rooms, guests, bookings, and payments efficiently. The system can keep track of room availability, record guest information, handle check-ins and check-outs, and generate bills. 


 1. Person
 Description: Base class for all people in the system.
 Attributes: name, email, phone
 Methods: Getters, setters, and toString() for displaying basic information.


 2. Guest (extends Person)
 Description: Represents a hotel guest. Can have bookings, car rentals, and
 restaurant orders.
 Attributes: customerId, address, lists of bookings, carRentals,
 restaurantOrders.
 Methods: Add bookings, rentals, orders; retrieve guest information.
 Static: noOfGuest – counts total guests.


 3. Employee (extends Person)
 Description: Represents a hotel employee.
 Attributes: employeeId, salary, position.
 Static: employeeCount – counts total employees.
 Methods: Terminate employee, getters/setters.


 4. Manager (extends Employee)
 Description: Manages hotel resources like rooms, cars, and staff. Can generate
 reports.
 Attributes: hotel (association to Hotel).
 Methods: Add/remove rooms and cars, view bookings and rentals, generate
 hotel reports.


 5. Receptionist (extends Employee)
 Description: Handles guest services like bookings, car rentals, and payments.
 Attributes: hotel
 Methods: Create bookings, rentals, process payments, update room/car
 availability.


 6. Housekeeping (extends Employee)
 Description: Cleans and maintains assigned rooms.
Attributes: assignedRooms (list of Rooms).
 Methods: Assign/unassign rooms, clean rooms.
 Room1.
 Description: Represents a hotel room.
 Attributes: roomId, type, price, isAvailable, list of housekeepers.
 Methods: Book, release, manage assigned housekeepers.


 7. Booking
 Description: Represents a room booking by a guest.
 Attributes: bookingId, guest, room, bookingPackage, startDate, endDate,
 totalPrice.
 Methods: Check-in/out, calculate total price, confirm booking.


 8. BookingPackage
 Description: Optional package for bookings (e.g., breakfast included).
 Attributes: packageId, name, description, includedServices,
 discountPercentage.
 Methods: Apply discount, check included services.


 9. Car
 Description: Represents a car available for rental.
 Attributes: carID, model, pricePerDay, isAvailable.
 Methods: Rent, return, getters/setters.


 10. CarRental
 Description: Represents a car rental for a guest.
 Attributes: rentalId, guest, car , startDate, endDate, totalPrice.
 Methods: Confirm rental, complete rental, calculate total, print details.


 11. Payment
 Description: Handles payment for bookings, car rentals, and restaurant orders.
 Attributes: paymentId, booking, carRentals, restaurantOrders, method, amount,
 invoiceDetails.
 Methods: Process payment, generate invoice.


 12. Invoice
Description: Represents a payment invoice for a guest.
 Attributes: invoiceId, issueDate, totalAmount, details, guest.
 Methods: Generate and display invoice.


 13. Hotel
 Description: Represents the hotel itself, using Singleton pattern.
 Attributes: name, location, lists of rooms, cars, bookings, carRentals,
 restaurant.
 Methods: Add rooms/cars/bookings/rentals, get available resources.28.


 14. Restaurant
 Description: Represents hotel restaurant.
 Attributes: name, availableItems, priceList, guests.
 Methods: Create and place orders, get menu.


 15. RestaurantOrder
 Description: Represents an order made by a guest in the restaurant.
 Attributes: guest, restaurant, orderedItems, totalPrice, orderDate.
 Methods: Add items, calculate total, order summary.44

 
 16. Room 
 Description: Represents a hotel room, managing its details, availability, and assigned  housekeepers.
 Attributes: roomId, type, price, isAvailable, housekeepers
 Methods: addHousekeeper, getHousekeepers, book, release, toString and setters/getters
